# 配置前端代理服务器绕开CORS拦截

## 问题分析
当前项目是纯前端项目，没有后端服务器，直接调用第三方API时遇到了浏览器的CORS拦截。为了解决这个问题，我需要创建一个简单的代理服务器来转发API请求。

## 解决方案
使用Node.js和Express创建一个简单的代理服务器，将前端的API请求转发到第三方API服务器，绕开浏览器的CORS拦截。

## 实现步骤

### 1. 创建代理服务器配置
- 创建`package.json`文件，添加必要的依赖
- 创建`proxy.js`文件，实现代理服务器逻辑

### 2. 代理服务器功能
- 监听本地端口（如3000）
- 转发所有API请求到第三方API服务器
- 添加CORS头，允许前端访问
- 处理请求参数和响应

### 3. 修改前端代码
- 更新`script.js`中的API请求地址，指向本地代理服务器
- 保持API请求的其他参数不变

### 4. 提供使用说明
- 如何安装依赖
- 如何启动代理服务器
- 如何使用代理服务器

## 技术栈
- Node.js
- Express
- http-proxy-middleware

## 预期效果
- 前端可以通过本地代理服务器访问第三方API
- 绕开浏览器的CORS拦截
- 保持原有的API请求逻辑不变

## 文件修改
- 创建`package.json`
- 创建`proxy.js`
- 修改`script.js`中的API请求地址

## 使用方法
1. 安装依赖：`npm install`
2. 启动代理服务器：`node proxy.js`
3. 启动前端服务器：`python -m http.server 8000`
4. 在浏览器中访问：`http://localhost:8000`

这样配置后，前端的API请求会通过本地代理服务器转发到第三方API，从而绕开浏览器的CORS拦截。
